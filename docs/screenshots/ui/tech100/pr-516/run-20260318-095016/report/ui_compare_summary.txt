TECH100 Portfolio redesign review
Run ID: 20260318-095016
PR: #516

Defects found in the current production page before redesign
- Layout hierarchy was still dominated by framed mini-modules instead of a single analytics workspace.
- The top summary treatment behaved like a model-card rail rather than a market-style overview ribbon.
- The main chart shared its row with a right-side matrix and factor rail, which diluted the hero surface.
- The heavy dark chart slab overpowered the rest of the page and left visually awkward empty depth under the canvas.
- Holdings, attribution, and sector content were split into cramped side columns instead of owning full-width surfaces.
- Methodology was fragmented across several small note boxes rather than attached contextually to the active analysis.
- The page still relied on many small button clusters, which made the interaction model feel busier than necessary.

What changed in this redesign
- Rebuilt the top of the route around a compact header and a rolling daily model tape fed by the latest live-backed portfolio summaries.
- Consolidated model, benchmark, range, metric, view, and attribution controls into a single workspace bar.
- Moved the main chart onto its own full-width row with lighter panel styling and a cleaner legend treatment.
- Replaced the old right-rail matrix/factor cards with one full-width comparison table plus a selected-model lens strip.
- Reworked holdings, attribution, and sector tilt into a tabbed full-width analytics explorer rather than split columns.
- Consolidated research inputs, control explanations, supported analytics, and deferred analytics into cleaner expandable method drawers.
- Tightened the full-history timeseries query to the six supported model codes so the page path no longer loads unused legacy series.
